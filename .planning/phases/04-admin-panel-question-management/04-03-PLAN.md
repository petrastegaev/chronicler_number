    Add static method `csv_import` to `QuestionService` in `backend/services/question_service.py`:
    - Signature: `async def csv_import(db: AsyncSession, file_content: bytes) -> dict`
    - Import `csv` and `io` at the top of the function (or at module level)
    - Use `csv.reader(io.StringIO(file_content.decode("utf-8-sig")))` for BOM-aware reading
    - Iterate rows with `enumerate(reader, start=1)`:
      - Skip empty rows (row is empty list or all whitespace after stripping)
      - If len(row) < 2: "Строка {row_num}: недостаточно полей"
      - Extract: `text = row[0].strip()`
      - If not text: "Строка {row_num}: Пустой текст вопроса"
      - Parse `answer_str = row[1].strip()` → try/except int conversion: "Строка {row_num}: Ответ не является целым числом"
      - Validate 0 <= answer <= 1_000_000: "Строка {row_num}: Ответ вне диапазона (0-1 000 000)"
      - Extract `category = row[2].strip() if len(row) > 2 and row[2].strip() else None`
      - If category and len(category) > 255: "Строка {row_num}: Категория слишком длинная"
      - On passing all validations: `await QuestionService.create(db, text=text, answer=answer, category=category)`, increment `added`
      - Collect errors in a list
    - Return `{"added": added, "errors": errors}`

    Add endpoint to `backend/routers/questions.py`:
    ```
    @router.post("/upload-csv", response_model=CsvImportResponse)
    async def upload_csv(file: UploadFile = File(...), db: AsyncSession = Depends(get_db)):
        content = await file.read()
        result = await QuestionService.csv_import(db, content)
        return CsvImportResponse(**result)
    ```
    - Import at top of file: `from fastapi import UploadFile, File`, `from schemas import CsvImportResponse`
    - The endpoint is at `/api/questions/upload-csv` (inherits the router prefix)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin browser -> CSV upload endpoint | File upload from admin panel |
| CSV file content -> backend parser | Untrusted file content enters Python csv module |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Denial of Service | POST /api/questions/upload-csv | accept | Large file upload could consume memory. FastAPI UploadFile uses spooled temp files. Acceptable risk for local WiFi booth with single admin user. ASVS V12 not applicable. |
| T-04-06 | Tampering | CSV content validation | mitigate | Backend validates every row: answer must be integer 0-1M, text non-empty, category max 255 chars. Invalid rows are skipped with descriptive errors. csv module handles quoting/escaping safely. No SQL injection risk (SQLAlchemy parameterized queries). |
| T-04-07 | Spoofing | CSV injection via question text | mitigate | React renders question text as string content, not dangerouslySetInnerHTML. Text content is rendered as plain text nodes, so formula injection (e.g., =A1) has no spreadsheet context. |
| T-04-SC | Tampering | npm/pip installs | mitigate | No new packages introduced. All dependencies already installed. |
</threat_model>
