      1. Store questions in self.questions, set self.state = "ready"
      2. Broadcast `game_started` event with player nicknames from manager:

    D. Method `async def _run_round(self, question)` — per D-11, D-01, D-04, D-13, D-15, D-17:
      1. Reset: self.p1_answer = None, self.p2_answer = None, self.state = "presenting_question"
      2. Broadcast `round_started`: `{"event": "round_started", "data": {"round_number": N, "total_rounds": 9, "question_text": question.text}}`
      3. Set self.state = "accepting_answers"
      4. Timer loop: for remaining in range(10, -1, -1):
         - Broadcast timer_tick: `{"event": "timer_tick", "data": {"remaining": remaining}}`
         - Store self._remaining = remaining (for reconnect state snapshot per D-06)
         - If remaining > 0: await asyncio.sleep(1)
         - NOTE: Use manager.broadcast() for all ticks (timer_tick goes to ALL connections including admin per D-17)
      5. After timer: await asyncio.sleep(0.05) — fairness window per Pitfall 2
      6. Set self.state = "showing_result"
      7. Compute result: call self._compute_round_result(question), append to self.round_results
      8. Broadcast `round_result` to ALL: manager.broadcast()
      9. Update scores: player1 gets +1 if winner=="player1", player2 gets +1 if winner=="player2", ties = 0 points (GAME-08)
      10. Send `score_update` to admin only: `{"event": "score_update", "data": {"player1_score": int, "player2_score": int, "round_number": int}}`
      11. await asyncio.sleep(3) — inter-round pause per D-15

      - if player_num == 1 and self.p1_answer is None: self.p1_answer = answer
      - elif player_num == 2 and self.p2_answer is None: self.p2_answer = answer
      - NOTE: answer is already validated (int 0-1,000,000) by the WebSocket dispatcher — this method only enforces first-wins.

      ```python
      def _compute_round_result(self, question) -> dict:
          correct = question.answer
          def diff(answer):
              if answer is None: return float("inf")
              return abs(answer - correct)
          d1 = diff(self.p1_answer)
          d2 = diff(self.p2_answer)
          if d1 < d2: winner = "player1"
          elif d2 < d1: winner = "player2"
          else: winner = "draw"
          return {
              "round_number": self.current_round,
              "correct_answer": correct,
              "player1_answer": self.p1_answer,
              "player2_answer": self.p2_answer,
              "winner": winner,
          }
      ```

      ```python
      async def _persist_game(self, winner_nickname: str | None):
          async with self.session_factory() as db:
              game_session = GameSessionModel(
                  player1_nickname=self.manager.player1_nickname or "",
                  player2_nickname=self.manager.player2_nickname or "",
                  player1_score=self.p1_score,
                  player2_score=self.p2_score,
                  winner_nickname=winner_nickname,
                  ended_at=datetime.now(timezone.utc),

              db.add(game_session)
              await db.flush()
              for r in self.round_results:
                  q = self.questions[r["round_number"] - 1]
                  round_record = Round(

                      question_id=q.id,

                      player2_answer=r["player2_answer"],

              stat = await db.execute(select(Stat).limit(1))

              await db.commit()
      ```

    Create `backend/game/tokens.py` with a module-level in-memory session token store for the JOIN-05 reconnect mechanism. Implementation per 02-RESEARCH.md Pattern 3 (lines 441-471) and Code Examples (lines 629-653):

    ```python
    import uuid

    # Module-level store: {token_hex: {"nickname": str, "role": str}}
    _reconnect_tokens: dict[str, dict] = {}

    def generate_token(nickname: str, role: str) -> str:
        """Generate a UUID hex token and store the session data."""
        token = uuid.uuid4().hex
        _reconnect_tokens[token] = {"nickname": nickname, "role": role}
        return token

    def restore_from_token(token: str) -> dict | None:
        """Look up session data by token. Returns None if not found or expired."""
        return _reconnect_tokens.get(token)

    def remove_token(token: str) -> None:
        """Remove a token from the store (e.g., on explicit disconnect)."""
        _reconnect_tokens.pop(token, None)
    ```

    Design rationale per RESEARCH:
    - UUID hex tokens = 32 chars, cryptographically random. Sufficient for booth-scale use (hundreds of reconnects max). No JWT library needed.
    - In-memory dict: lost on server restart, but that is acceptable for a booth demo (A3 in Assumptions Log).
    - `remove_token()` added for cleanup on explicit disconnect — not in the original RESEARCH examples but needed to prevent memory leaks across many restarts.
    - No async needed — this is a synchronous dict access, called from the event loop.
  </action>

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebSocket client -> Server | Player/admin sends JSON messages. GameSession processes only validated data from the WebSocket dispatcher. |
| GameSession in-memory -> SQLite | Game results persisted at game end. Protected by asyncio.shield(). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | GameSession.submit_answer() | mitigate | First-answer-wins (D-12): subsequent submissions from the same player in the same round are silently ignored. State check ensures answers only accepted during `accepting_answers` phase. |
| T-02-02 | Information Disclosure | Answer secrecy | mitigate | GameSession NEVER broadcasts individual answers. Both answers stored in `self.p1_answer` / `self.p2_answer` instance variables. Only `round_result` event (after timer expiry) contains both answers (GAME-04, D-13). |
| T-02-03 | Denial of Service | Timer manipulation | mitigate | Server-authoritative timer via `asyncio.sleep(1)` loop (D-01, D-05). Client cannot influence timer speed or duration. |
| T-02-04 | Tampering | Late answer injection | mitigate | `submit_answer()` checks `self.state == "accepting_answers"` before accepting. After state transitions to `showing_result`, all submissions silently ignored (D-12). |
| T-02-05 | Tampering | Task cancellation kills persistence | mitigate | `asyncio.shield()` wraps `_persist_game()` call (Pitfall 3). Survivies task cancellation from WebSocket disconnect. Single atomic transaction: GameSession + 9 Rounds + Stat. |
| T-02-06 | Denial of Service | Admin restart double-tap | mitigate | `reset_game()` is idempotent — checks `active_session is None` or `state == "waiting_for_players"` before proceeding. Second restart call is a no-op (handled in 02-02 main.py but session state is set in 02-01). |
| T-02-07 | Spoofing | Session token guessing | accept | UUID v4 hex tokens (32 chars, 122 bits entropy). For local WiFi booth with <100 reconnect attempts, collision odds are negligible. In-memory store, never persisted or logged. |

</threat_model>
