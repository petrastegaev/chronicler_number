FIX D (P1 — BUG-FONT-MISMATCH): The CoFo Sans .otf files are mode 0600 (rw-------) owned by root in the image, but the app runs as non-root `appuser` (Dockerfile line 25 `USER appuser`). appuser cannot read them → StaticFiles sends the content-length header from stat() but 0 body bytes → browser aborts with ERR_CONTENT_LENGTH_MISMATCH. Inter .woff2 are 0644 and work. Two-part fix (defense in depth):
  (a) In the Dockerfile, AFTER the `COPY --from=frontend-build /app/dist ./static` line (line 17) and BEFORE `USER appuser` (line 25) — placing it after the existing `RUN mkdir -p /app/data && chown appuser /app/data` is fine — add a new line: `RUN chmod -R a+rX ./static`. (`a+rX` grants read to all and execute/traverse only on directories — correct for a static asset tree; do not use `a+rwx`.)
  (b) Fix the SOURCE files so they are correct regardless of build context: `chmod 644 backend/static/fonts/CoFoSansPixel.otf backend/static/fonts/CoFoSansRegular.otf frontend/public/fonts/CoFoSansPixel.otf frontend/public/fonts/CoFoSansRegular.otf`. Use the Bash tool for the chmod.
  (Optional cleanup, harmless either way: delete the stray `frontend/src/assets/fonts/__MACOSX/` resource-fork directory.)
